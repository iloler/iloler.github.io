春去秋来，寒来暑往，时光流走，四季更替。距离上次更新博客，竟已过去3年多。
这次疫情倒是让原本紧张的生活节奏，略微降了下来。于是乎想起了我在几年前，还是个小萌新的时候，雄心勃勃要坚持写博客。还好因为穷，买不起新的MacBook，在“访达”的角落里找到了名为blog的文件夹。折腾了几天，总算复活了个人博客。想了想，以后没准还会折腾一遭，遂趁机做个记录，总结一下。

框架

hexo

hexo是一个快速、简洁且高效的博客框架。当初为什么选择它当博客框架，有点记不清了，可能是当时看了开发者头条推荐的文章。

安装

安装hexo依赖nodejs，git。此二者安装不再赘述。
通过以下命令安装hexo。
npm install -g hexo-cli

新建

安装完毕之后，通过以下命令在当前目录下新建项目。
hexo init blog-demo
cd blog-demo
npm install
执行命令完毕后，用vs code打开，可以看到目录结构如图所示。

_config.yml

其中_config.yml是配置文件，我们可以配置很多信息。
比如我把title设置成“养只阿柴捏脸玩”，那么打开博客，映入眼帘的是大大的“养只阿柴捏脸玩”几个字。
因为我用的主题是jacman，所以theme设置为“jacman”。
比较重要的是deploy的配置，因为我是发布到github上，所以配置是这样的。
deploy:
type: git
repository: git@github.com:chenpeng/chenpeng.github.io.git
branch: master

package.json

package.json是node的包文件，依赖的包都在里面。
因为我的博客是从语雀同步过来的，所以这里需要加上语雀的配置。
"yuqueConfig": { "postPath": "source/_posts/yuque", "cachePath": "yuque.json", "mdNameFormat": "title", "adapter": "hexo", "concurrency": 5, "baseUrl": "https://www.yuque.com/api/v2", "login": "yangzhiachainielianwan", "repo": "lvaydp", "onlyPublished": false, "token": "xxxxxxxxxxxxxxxxxxx" }

scaffolds

scaffolds是模版文件夹，提供了几个模版用来快速生成文章。
有需要的话，也可以自己搞几个模版留着用。

source

source是资源文件夹，除了posts文件夹，其他以（下划线）开头或者隐藏的文件夹会被忽略。Markdown 和 HTML 文件会被解析并放到public文件夹，其他文件会被拷贝过去。
因为有插入图片的需求，我在这个目录下建了一个“images”文件夹，用来存放图片。

themes

themes是主题文件夹，我们可以自由选择自己喜欢的主题，安装到这里。
我用的主题是jacman，作者照片，博客logo等可以在themes/jacman/source/img目录下替换相应的图片。更多详情参考Jacman。

yuque-hexo

安装

npm i -g yuque-hexo

同步

yuque-hexo sync

清除

yuque-hexo clean
语雀同步的文章会在博客项目目录下生成两份文件。

  • yuque.json: 从语雀 API 拉取的数据
  • source/_posts/yuque/*.md: 生成的 md 文件

    配置

    需要在package.json增加配置，已在前文讲述，其中token需要在语雀后台生成。

编辑器

技术人写博客，倾向于使用markdown。我先后使用MWeb,VS code，不过老实说手写markdown十分蛋疼。最后选择语雀来写，通过同步工具同步到GitHub上。怎一个爽字了得！

服务器

Github Pages

因为博客都是静态的，所以也不需要额外的服务器。GitHub pages可以满足需求。
GitHub账号注册之类的琐事就不赘述了。登录GitHub之后，我们需要在Repositorys新建一个Repository，命名为：用户名.github.io。新建好之后，点击Repository的Settings。一直往下拖动，拖到GitHub pages相关的地方，设置一下即可。

此时通过http://chenpeng.github.io/就可以访问到我的博客了。

Github Action

经常手动同步，也很麻烦啊！作为一个懒人，我可不想手动搞。刚好GitHub Action开始公测，申请之后就可以用了。话说这玩意就是个docker，理论上有无限可能，做一下同步简直小菜一碟。
我们把blog-demo项目上传到github上，仓库设置为Private。

点击“New workflow”按钮，编辑yml文件。我希望在push的时候，能自动同步语雀的文章。并且，希望每天定时同步。那么，我需要的yml文件可以这么写。

可以看到，我这个任务名字叫“Deploy Blog”。当“push”时，出发任务。并且有个定时执行的操作。
再看看“jobs”，跑在最新的ubuntu系统上。第一步先“Checkout”master分支，后面使用chenpeng/deploy-hexo-sync-yuque-action@master。
这是个什么呢？这其实是dockerfile，可以自己根据需要编写，有了它想咋玩咋玩。最主要的是entrypoint.sh文件，我们来看一看。

一目了然看出了，其实就是检出blog-demo代码，然后安装了npm包，同步一下语雀的文章，再提交一下。
完事，收工！
